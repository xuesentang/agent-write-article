# 图文合并问题修改文档

## 问题描述

最终输出的文章仍然是 Markdown 格式，图片只显示为链接文字而非真实图片。用户期望得到一个**图文并茂的富文本文章**，而非 Markdown 源码。

## 根因分析

### 核心问题：整个链路中不存在"Markdown → 富文本/HTML"的转换环节

追踪完整数据流：

```
ContentAgent 生成 Markdown 正文（含 IMAGE_PLACEHOLDER 占位符）
    ↓
ImageAnalyzerAgent 解析占位符，生成图片任务列表
    ↓
ImageGeneratorAgent 生成图片 → 上传COS → 替换占位符为 ![配图](COS_URL)
    ↓
task.py 将 mergedContent 存入 article.final_output（仍是 Markdown！）
    ↓
前端 Result.vue 用 markdown-it 渲染：v-html="renderMarkdown(final_output)"
```

**问题定位**：

| # | 环节 | 当前行为 | 问题 |
|---|------|----------|------|
| 1 | `_merge_images_into_content`（`image_generator_agent.py:579`） | 将占位符替换为 `![配图](URL)` | 输出仍为 Markdown 图片语法 |
| 2 | `task.py:517` | 将 mergedContent 存为 `final_output` | final_output 是 Markdown 字符串 |
| 3 | 前端 `Result.vue:268` | `renderMarkdown(final_output)` 用 markdown-it 渲染 | **图片应能正常显示**，但用户反馈看不到图片 |

### 深入分析：为什么 markdown-it 渲染后图片仍不可见？

重新审视后，`markdown-it` 会将 `![配图](URL)` 正确渲染为 `<img src="URL">`。前端 CSS 中也有 `:deep(.markdown-content img)` 样式。**所以图片应该是能显示的**，但用户说"只有图片链接，没有真实图片"。

这说明真正的问题可能是：

1. **占位符替换失败** — `_merge_images_into_content` 的正则没有匹配到正文中的占位符，导致 `mergedContent` 中仍然包含原始的 `![IMAGE_PLACEHOLDER](image_N|关键词)` 文本
2. **final_output 未正确存储/传输** — SSE 的 done 事件或 API 返回时 final_output 为空或未更新
3. **前端取错字段** — 前端可能优先展示了 `content` 字段（含未替换占位符的原文），而非 `final_output`

让我验证这些假设：

### 假设1验证：占位符替换正则

```python
# image_generator_agent.py:575
pattern = rf'!\[IMAGE_PLACEHOLDER\]\({re.escape(result.placeholderId)}\|[^)]*\)'
```

正文中的占位符格式：`![IMAGE_PLACEHOLDER](image_1|自媒体、内容创作、数字化时代)`

result.placeholderId 值为 `image_1`，正则变为：
```
!\[IMAGE_PLACEHOLDER\]\(image_1\|[^)]*\)
```

**这个正则应该能匹配**。但存在一个潜在问题：如果正文中占位符的格式与预期不完全一致（如多出空格、换行等），就会匹配失败。代码中有宽松匹配的回退逻辑，所以可能性不大。

### 假设2验证：final_output 存储链路

在 `task.py:516-518`：
```python
await article_repo.update_final_output(
    article_id=article.id,
    final_output=generator_output.mergedContent,
)
```

同时 `task.py:511-515` 还更新了 content：
```python
await article_repo.update_content(
    article_id=article.id,
    content=generator_output.mergedContent,
    word_count=str(len(generator_output.mergedContent)),
)
```

**两个字段都被更新为合并后的内容**，这条路看起来没问题。

### 假设3验证：前端取值

```javascript
// Result.vue:266-268
v-if="article.final_output || article.content"
v-html="renderMarkdown(article.final_output || article.content || '')"
```

前端优先取 `final_output`，如果没有则取 `content`。**如果后端正确更新了这两个字段，前端应该能拿到合并后的内容。**

### 最终诊断

综合分析，问题最可能出在以下**两个层面的组合**：

1. **后端层面**：占位符替换后的 `mergedContent` 仍为 Markdown 格式。当 `markdown-it` 渲染 `![配图](COS_URL)` 时，如果 COS URL 有跨域或访问权限问题，图片加载会失败。但如果 URL 可访问，图片应正常显示。

2. **用户期望层面**：用户明确说"不要 markdown 格式"，想要"正常的图文兼备的文章"。这意味着用户希望**导出的结果不是 .md 文件，而是类似 Word/PDF/HTML 等可以直接阅读的富文本格式**。当前导出功能（`Result.vue:83`）直接导出 Markdown：
   ```javascript
   const blob = new Blob([`# ${data.title}\n\n${data.content}`], { type: 'text/markdown' })
   a.download = `${data.title}.md`
   ```

**结论**：问题不是图片无法显示，而是整个应用缺乏将 Markdown 转换为富文本格式（HTML/Word/PDF）的能力。最终输出始终是 Markdown 字符串，无论前端怎么渲染，导出的文件始终是 .md。

---

## 修改方案

### 方案总览

分为 **3 个修改层次**，按优先级排列：

| 优先级 | 修改 | 说明 |
|--------|------|------|
| P0 | 后端新增 HTML 转换层 | 将 Markdown + 图片转为 HTML，存入新的 `final_html` 字段 |
| P1 | 前端结果页展示 HTML | Result.vue 和 Create.vue 优先展示 HTML 富文本 |
| P2 | 导出功能增强 | 支持导出 HTML/PDF/Word，而非仅 Markdown |

```python
def _convert_markdown_to_html(self, markdown_content: str) -> str:
    """
    将 Markdown 内容转换为 HTML 富文本

    Args:
        markdown_content: Markdown 格式内容（图片占位符已替换）

    Returns:
        HTML 富文本内容
    """
    import markdown
    from markdown.extensions.tables import TableExtension
    from markdown.extensions.fenced_code import FencedCodeExtension

    # 配置 markdown 扩展
    extensions = [
        'tables',
        'fenced_code',
        'codehilite',
        'toc',
        'nl2br',  # 换行转 <br>
    ]

    extension_configs = {
        'codehilite': {
            'css_class': 'highlight',
            'linenums': False,
        },
    }

    html_content = markdown.markdown(
        markdown_content,
        extensions=extensions,
        extension_configs=extension_configs,
    )

    return html_content
```

```python
async def _send_all_complete_event(
    self, task_id: str, results: List[ImageResult], 
    merged_content: str = "", merged_html: str = ""
):
    """
    发送所有图片完成事件

    Args:
        task_id: 文章生成任务 ID
        results: 图片结果列表
        merged_content: 图文合并后的 Markdown 正文
        merged_html: 图文合并后的 HTML 富文本
    """
    success_count = sum(1 for r in results if r.status == ImageTaskStatus.COMPLETED)
    failed_count = sum(1 for r in results if r.status == ImageTaskStatus.FAILED)

    await sse_manager.send_event(
        task_id=task_id,
        event_type=SSEEventType.IMAGE_ALL_COMPLETE,
        stage=SSEStage.IMAGE,
        data={
            "task_id": task_id,
            "total_count": len(results),
            "success_count": success_count,
            "failed_count": failed_count,
            "results": [
                {
                    "placeholderId": r.placeholderId,
                    "url": r.url,
                    "source": r.sourceProvider.value,
                    "status": r.status.value,
                }
                for r in results
            ],
            "merged_content": merged_content,
            "merged_html": merged_html,  # 新增：携带 HTML 富文本
        },
        progress=80,
        message=f"图片生成完成: {success_count} 成功, {failed_count} 失败",
    )
```

```python
@router.get("/articles/{article_id}/export", summary="导出文章")
async def export_article(
    article_id: str,
    format: str = "html",  # 默认改为 HTML
    db: AsyncSession = Depends(get_db),
):
    """
    导出文章

    - **article_id**: 文章唯一标识
    - **format**: 导出格式 (html/markdown)，默认 html
    """
    repo = ArticleRepository(db)
    article = await repo.get(article_id)

    if not article:
        return ApiResponse.error(code="1300", message="文章不存在")

    if format == "html" and article.final_html:
        return ApiResponse.ok(
            data={
                "title": article.selected_title,
                "content": article.final_html,
                "html": article.final_html,
                "word_count": article.word_count,
                "format": "html",
            },
        )

    # Markdown 导出
    if article.final_output:
        return ApiResponse.ok(
            data={
                "title": article.selected_title,
                "content": article.final_output,
                "word_count": article.word_count,
                "format": "markdown",
            },
        )

    return ApiResponse.ok(
        data={
            "title": article.selected_title,
            "content": article.content,
            "images": [ImageInfo(**img) for img in (article.images or [])],
            "word_count": article.word_count,
            "format": "markdown",
        },
    )
```

```typescript
    h2 { font-size: 24px; margin: 20px 0 12px; }
    h3 { font-size: 20px; margin: 16px 0 8px; }
    img { max-width: 100%; height: auto; border-radius: 8px; margin: 16px 0; display: block; box-shadow: 0 2px 8px rgba(0,0,0,0.1); }
    p { margin: 12px 0; }
    ul, ol { padding-left: 24px; }
    blockquote { border-left: 4px solid #ddd; padding-left: 16px; color: #666; margin: 16px 0; }
    code { background: #f5f5f5; padding: 2px 6px; border-radius: 3px; }
    pre { background: #f5f5f5; padding: 16px; border-radius: 8px; overflow-x: auto; }
  </style>
</head>
<body>
  <h1>${data.title}</h1>
  ${data.html}
</body>
</html>`
      const blob = new Blob([fullHtml], { type: 'text/html' })
      const url = URL.createObjectURL(blob)
      const a = document.createElement('a')
      a.href = url
      a.download = `${data.title}.html`
      document.body.appendChild(a)
      a.click()
      document.body.removeChild(a)
      URL.revokeObjectURL(url)
    } else {
      // 导出 Markdown
      const blob = new Blob([`# ${data.title}\n\n${data.content}`], { type: 'text/markdown' })
      const url = URL.createObjectURL(blob)
      const a = document.createElement('a')
      a.href = url
      a.download = `${data.title}.md`
      document.body.appendChild(a)
      a.click()
      document.body.removeChild(a)
      URL.revokeObjectURL(url)
    }

    message.success('文章已导出')
  } catch (err: any) {
    message.error(err.message || '导出失败')
  }
}
```
